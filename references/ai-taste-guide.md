# AI 味清理指南

本文件定义完成 A/B/C 三个文本文件后的**主动扫描 + 强制重写**流程。

这不是"检查有没有问题"，而是逐条扫描全文，命中的必须当场重写，不允许"未发现问题"作为通过答案。每一条都要报告：**扫描结果 → 命中行 → 原句 → 改句**。

完成中文版清理后，执行英文版清理（见本文最后一节）。

---

## 中文版扫描（逐条执行，顺序不可跳过）

### 扫描 1：开场引用块

检查文章第一个 `>` 引用块是否包含以下任意一种：
- 破折号 `——`
- "不是……而是……" / "不仅仅是……更是……" / "并非……而是……"

命中 → 重写为**直接陈述核心判断**，不用转折句式，不用破折号。

> 开场第一句是读者的第一印象，也是 AI 痕迹最暴露的位置。

**示例：**
- ❌ `> 争论的不是这笔生意好不好——而是它"好"的时间窗口还有多久。`
- ✅ `> Venture Global 建起了一门真实的生意。接下来的问题只有一个：这台套利机器还能跑多久。`

---

### 扫描 2：全文破折号计数

统计全文 `——` 出现次数。超过 4 次 → 从最密集的段落开始，将多余的改为句号（拆成两句）或逗号，直到总数 ≤ 4。

**重写模式：**
- `"A——B"` → B 是解释 A 时：改为 `"A。B"` 或 `"A，B"`
- `"在表面上是 X——在本质上是 Y"` → `"本质是 Y，X 只是外衣"`
- `"A——这意味着 B"` → `"A，这意味着 B"` 或 `"A。B"`

删除信号：如果破折号两侧任意一半单独成立，这个破折号大概率可以去掉。

---

### 扫描 3："不是/并非……而是……"句式

全文搜索（不限于开场）：
`不是……而是`、`不仅仅是……更是`、`并非……而是`、`不只是……更是`

命中 → 删掉否定前半句，直接说肯定结论。

**示例：**
- ❌ `护城河不是技术，而是执行壁垒`
- ✅ `护城河的本质是执行壁垒`

---

### 扫描 4：空洞导入词

全文搜索以下词语（段首出现时尤其警觉）：
`值得注意的是`、`值得警惕`、`此外`（段首）、`与此同时`（段首）、`综上所述`、`总而言之`、`由此可见`、`不难看出`、`不可否认`

命中 → 直接删掉，后面的内容成为独立句子。

---

### 扫描 5：AI 套话词

全文搜索：
`深耕`、`赋能`、`致力于`、`构筑`、`彰显`、`重塑`、`引领`、`打造`、`生态闭环`、`全方位`、`无缝`

命中 → 替换为具体动词或具体描述。**不允许用同义套话替换**——必须说清楚这个公司实际做了什么。

**示例：**
- ❌ `深耕 LNG 市场`
- ✅ `在 LNG 市场连续投入了三个项目，总产能超过 6000 万吨/年`

---

### 扫描 6：段落结构检查

抽查全文 3-5 个段落，识别以下 AI 结构模式：
- **首先/其次/再次/最后** 串联多段（超过两段时机械感强）
- **每段末尾都有总结句**（"这说明……"/ "可以看出……"）
- **粗体滥用**（超过 5 处粗体且不都是关键数字/术语）

命中 → 合并段落、删掉总结句、减少粗体。

---

## 英文版扫描

中文版清理完成后，对英文版 `[slug].en.md` 执行以下扫描：

### 禁用词检索

逐一搜索，命中必须替换：

| 禁用 | 替换方向 |
|------|---------|
| `delve into` | `examine` / `look at` / `break down` |
| `testament to` | 删掉，直接描述事实 |
| `underscore` | `show` / `reveal` |
| `nuanced` | 删掉，直接描述细节 |
| `pivotal` | `key` / `critical`，或不加形容词 |
| `showcase` | `show` / `demonstrate` |
| `foster` | `build` / `drive` |
| `leverage`（动词） | `use` / `draw on` |
| `cornerstone` | `foundation` / `core` |
| `unprecedented` | 提供数据，不用形容词 |
| `It's worth noting` | 删掉，直接陈述 |
| `It's important to note` | 同上 |

### 禁用句式检索

- `Not only X, but also Y` → 直接说 Y 的数据
- `In conclusion` / `In summary` / `Overall` / `To sum up` → 删掉
- `From X to Y`（两端语义不相关时）→ 重构

### 节奏检查

检查是否有连续 3 段以上以 `Firstly` / `Moreover` / `Furthermore` / `Additionally` 开头，命中 → 删掉这些连接词，段落直接开始。
